## 식별자 네이밍 규칙
- 식별자에는 특수문자를 제외한 문자, 숫자, 언더스코어, $를 포함할 수 있다.
- 식별자는 언더스코어, 달러기호, 문자로 시작해야 한다.
- 예약어는 식별자로 사용할 수 없다.

## 네이밍 컨벤션
```javascript
// camel case
const firstName;

// snake case
const first-name;

// pascal case
const FirstName;

// hungarian case
const strFirstName;
```
- 일관성을 유지한다면 어떤 네이밍 컨벤션을 사용해도 무관하지만, 통상적으로는 
    - 변수나 함수 - 카멜 케이스 
    - 생성자 함수, 클래스 - 파스칼 케이스
    를 사용한다.
